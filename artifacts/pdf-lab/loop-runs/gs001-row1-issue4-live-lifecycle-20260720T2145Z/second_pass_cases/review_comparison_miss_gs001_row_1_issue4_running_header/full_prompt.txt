- original page image path
- annotated page image path or crop image path
- actual deterministic extraction JSON, or actual candidate extractions linked from the deterministic output for expected-side comparison misses

- bbox, page, element ID, and provenance paths supplied in the payload

Technique evidence means generic knowledge about PDFs, tables, charts, captions, equations, or extraction heuristics. Technique evidence may explain validation rules, but it may not justify a human triage card by itself.

Closed decision vocabulary:
- `agent_resolved`
- `human_triage`
- `fixture_candidate`
- `blocker`

Human triage rule:
Return `human_triage` only when the supplied artifacts contain a concrete unresolved conflict that cannot be resolved by the selected preset validators.

Expected-side comparison miss rule:
If `actual_json` is null but `actual_candidates[]` is non-empty, treat `actual_candidates[]` as deterministic extraction evidence. Do not return blocker solely because `actual_json` is null in that case.

Table-specific hard rule:
If the visible evidence shows a table header plus one data row and the actual JSON captures that visible table object, do not ask whether it is a real table. Return `agent_resolved` unless the payload proves visible content is cut off or candidate corrected JSON conflicts with actual JSON.

Table-merge hard rule:
For table merge, continuation, rowspan, colspan, or split-table questions, use the attached annotated page images and `table_merge_review` metrics together. Do not ask a human whether two table regions should merge unless:
- both relevant page/table images are supplied or explicitly unavailable,
- extracted shape/data-quality metrics are supplied for the primary and adjacent table candidates,
- the selected table preset validators cannot determine whether this is one logical table, separate tables, or an extractor/preset backlog item.
If metrics and images are sufficient, return `agent_resolved` or `fixture_candidate` with a deterministic engine/preset recommendation.

Output rule:
Return only one JSON object. Do not return markdown, commentary, headings, or prose outside JSON.
